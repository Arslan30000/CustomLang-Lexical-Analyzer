================================================================================
                        LANGUAGE LEXICAL SPECIFICATION
================================================================================

1. KEYWORDS
   The following words are reserved and cannot be used as identifiers:
   - start, finish
   - declare, input, output
   - if, else
   - loop, break, continue
   - function, return, condition

2. IDENTIFIERS
   - Format: Must start with an Uppercase letter [A-Z].
   - Followed by: Any combination of letters [a-zA-Z], digits [0-9], or underscore [_].
   - Constraint: Maximum length is 31 characters.
   - Regex: [A-Z][a-zA-Z0-9_]{0,30}

3. LITERALS
   a) Integer Literals
      - Sequence of digits.
      - Can be optionally signed (+/-).
      - Regex: [+-]?[0-9]+

   b) Floating Point Literals
      - Digits containing a decimal point.
      - Optional exponent part (e or E).
      - Constraint: Max 6 digits after the decimal point (precision check).
      - Regex: [+-]?[0-9]+\.[0-9]{1,6}([eE][+-]?[0-9]+)?

   c) Boolean Literals
      - Values: true, false

   d) String Literals
      - Enclosed in double quotes (").
      - Can contain spaces and special characters.
      - Regex: "[^"\n]*"

   e) Character Literals
      - Enclosed in single quotes (').
      - Represents a single character or escape sequence.
      - Regex: '([^'\\\n]|\\.)'

4. COMMENTS
   a) Single-Line Comment
      - Starts with '##' and continues to the end of the line.
      - Regex: ##.*

   b) Multi-Line Comment
      - Starts with '#|' and ends with '|#'.
      - Can span multiple lines.

5. OPERATORS & PUNCTUATORS
   a) Arithmetic: +, -, *, /, %, **, ++, --
   b) Relational: <, >, <=, >=, ==, !=
   c) Logical:    &&, ||, !
   d) Assignment: =, +=, -=, *=, /=
   e) Punctuators: ( ) { } [ ] , ; :

6. WHITESPACE
   - Spaces, Tabs (\t), Newlines (\n, \r) are skipped by the scanner unless inside a string.
